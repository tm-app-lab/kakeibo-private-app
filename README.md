# 家計・給与管理アプリ

このアプリは、給与管理と支出項目管理を同じ本体画面で扱う個人用ローカルWebアプリです。

## 正しい起動方法

通常は `家計アプリを開く.cmd` を実行して `index.html` を開きます。

現行の起動経路は以下です。

1. `index.html`
2. `data/household-data.js`
3. `data/payroll-data.js`
4. `storage.js`
5. `ui.js`
6. `household.js`
7. `import.js`
8. `payroll.js`
9. `app.js`

## 統合方式

収入管理は iframe ではなく、本体アプリ内のネイティブ画面として統合しています。

- `index.html` の `#panel-income` は空の表示領域です。
- 収入管理画面は `payroll.js` の `payrollRenderShell()` から本体DOMへ描画されます。
- 給与の初期データは `data/payroll-data.js` の `window.PAYROLL_DATA` を参照します。
- 支出管理は `household.js`、外部CSV取り込みは `import.js` が担当します。
- `app.js` は初期化とイベント接続を担当します。

## archive について

`archive/legacy-income-integration` は旧iframe統合・旧単体給与アプリの参照用保管場所です。

現行アプリは archive 内のHTMLやJavaScriptを読み込みません。通常利用では archive 内のファイルを開かないでください。

## データ保存

アプリの編集データは、この端末のブラウザ内の `localStorage` に保存されます。

端末変更やブラウザ初期化に備え、収入管理・支出管理それぞれのバックアップ機能を使って定期的に控えを作成してください。

## 手動テスト観点

変更後は、最低限以下を確認します。

### 共通

1. `家計アプリを開く.cmd` から起動できる。
2. 左メニューで `収入管理` と `支出管理` を切り替えられる。
3. ヘルプを開閉できる。
4. 設定内の `バックアップ` からJSONを出力できる。
5. 設定内の `復元` でJSONを読み込み、不正JSONでは既存データが壊れない。
6. 画面を再読み込みしても、保存済みデータが残っている。

### 収入管理

1. ユーザーを切り替えられる。
2. 登録年月を入力して、写真読み込みまたは手入力へ進める。
3. 入力フォームの下にあるサマリーを開くと、手取り、総支給、控除、前月差のKPIが表示される。
4. 残業依存度、実質時給、実効税率のKPIが表示される。
5. 手入力中にKPIと状態表示がリアルタイム更新される。
6. 状態表示が `未保存`、`保存済`、`変更あり` のいずれかになる。
7. 入力方法は `写真から読込`、`前月コピー`、`手入力`、`差分入力` の順に表示される。
8. `前月コピー` 後、通常の手入力として修正、確認、保存できる。
9. 前月データがある登録年月では `差分入力` が押せる。
10. `差分入力` では各入力欄の横に前月値と差分額が表示され、前月と異なる行が強調される。
11. 差分入力後も通常の保存データとして、確認、保存できる。
12. 保存時に給与データの自動バックアップが作成される。
13. データ管理に、安全確認エリア、最終保存、最終バックアップ、スナップショット件数、直近スナップショットが表示される。
14. 保存・削除・復元前に直近5件までのスナップショットが作成される。
15. スナップショットから復元できる。
16. バックアップ復元前に現在データが自動退避され、不正データでは復元されない。
17. 保存後に、登録年月、手取り、前月比、総支給、控除の完了メッセージが表示される。
18. 入力画面の手取りグラフを開くと、直近12か月の手取り推移が表示される。
19. 手入力で給与項目を入力し、確認後に保存できる。
20. 保存済み年月を再表示できる。
21. タブが `入力`、`分析`、`データ管理` の3つに整理されている。
22. `分析` タブを開くと、最新月の手取り、年間累計、前年同月比の要約だけが表示される。
23. `分析を開く` で、詳細グラフ、年間集計、Top5、年別比較を確認できる。
24. `分析を閉じる` で、詳細分析エリアを折りたためる。
25. データ管理ではExcel出力と直近データ復元を確認でき、バックアップと復元は設定内に集約されている。

### 支出管理

1. 支出項目の行をクリックすると、詳細カードに内容が表示される。
2. `新規` で新しい支出項目を作成できる。
3. 編集モードで内容を変更し、`決定` と `キャンセル` が動く。
4. 支払者、種別、支払い方法、固定/変動、支払い時期などのプルダウンが表示される。
5. 一覧のソート、検索、表示フィルターが動く。
6. 設定画面でプルダウン候補を確認できる。

### 外部取り込み

1. マネーフォワードCSVと楽天カードCSVを選択して取り込める。
2. 同じ年月のCSVを再取り込みしても、同一月データとして上書き・整理される。
3. 年月を切り替えて取り込み済み明細を閲覧できる。
4. マネーフォワードと楽天カードのサブタブを切り替えられる。
5. マネーフォワード明細の `紐づけ` から楽天カード明細または同月タブへ移動できる。
6. 編集モードで個別削除と一括削除の表示が切り替わる。

## 自動テスト

CSV取り込みの基礎ロジックは、以下で確認できます。

```powershell
node tests\csv-import.test.js
```

確認対象は、CSV種別判定、マネーフォワード行の正規化、楽天カード行の正規化、空行除外、同じ年月CSVの二重登録防止です。

## 簡易ログイン

このアプリは、個人利用向けの簡易パスワード画面を表示します。

- パスワード定数は `auth.js` の `APP_PASSWORD` で変更できます。
- 認証済みフラグは `localStorage` の `household_app_auth_passed` に保存されます。
- ログアウトしても、収入・支出・外部データなどの家計データは削除されません。

### セキュリティ上の注意

この認証は、URLを知っているだけの人にアプリ画面を見せにくくするための簡易ガードです。
GitHub Pages上のHTML/JavaScriptは閲覧可能なため、技術的にはパスワードを解析され得ます。
本格的に非公開化する場合は、サーバー側認証や認証付きホスティングを使ってください。
